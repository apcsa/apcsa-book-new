New Terms:

  • Conditional Statement
  • Comparison Operator
  • Bug
  • Logic Error
  • Boundary Case

Key Concepts:

  • Conditional statements let your code make decisions based on user input or other run-time information.
  • The first principle of program testing is that your testing must try out every statement in the program, since any statement that isn't tested may contain a bug.
  • The second principle of program testing is that your tests must check every potential path through conditional statements in your code.
  • A boundary case is a situation that tests your code in an exagerated way. For instance, by using unusually large or small values for a parameter.
