The next three weeks of activities all involve working with the GridWorld program supplied by AP CS. GridWorld comes with a writeup, called a "case study", that describes the features of the program and guides the reader's exploration of the program.

Activities will mainly involve analyzing and modifying program segments, as in the Monty Hall and sound sampling lessons. GridWorld differs from those other programs, however, in the amount and complexity of code you'll be studying. It involves several interacting classes, totaling around five hundred lines of code. Tasks such as predicting program behavior and making sure that modifications don't have any undesirable side effects will pose more of a challenge in this setting.

Another interesting aspect of this program is its heavy use of inheritance. GridWorld provides an Actor class that can move and "act". Almost all the activities involve designing or analyzing classes that set up different kinds of actors via inheritance from the Actor class.

The first week of activities is based on parts 1 and 2 of the case study. We will set up GridWorld in the next step. Away we go!
