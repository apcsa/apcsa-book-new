Here are all the method headers, or the first line of their definitions. (You can also call this the method's signature). 

   public void eat (int amount)
   public void refill (int amount)
   public int candyCount ( )
   public CandyJar (int startingCount)

The header for each method contains four important pieces of information:

  • The method's accessibility (public or private, according to whether or not something in another class is allowed to call it).
  • The kind of value it returns. void means it doesn't return any information, while int means that it will return an integer. And, constructors are special, and don't specify a return type.
  • The name of the method. Again, constructors are special: the name of a constructor is the same as the name of the class.
  • The types and names for the method's parameters, which specify how information is given to the method (and, this means, how information is given to the object with those methods).

    The parameters in the eat and refill methods tell the method how much should be taken away or added to the CandyJar's instance variable. For the constructor, this parameter specifies how much candy should be in the jar when it is created. And the poor candyCount method doesn't need any parameters to do its job.
