private int myCandyCount;  //the instance variable

The above line, the declaration for the myCandyCount instance variable, contains two other pieces of information.


First is its accessibility. The myCandyCount variable shouldn't be accessed directly by outsiders, so it has private accessibility. This will be the case for almost every instance variable we encounter.

(Classes also have accessibility. The CandyJar class is public, which means that outsiders can work with CandyJar objects.)


The other information in the declaration is the type of the variable. Here, that's int, meaning that we are representing the contents of the CandyJar object as a whole number, or an integer.

This was a design decision. A CandyJar object will be able to hold 27 units, but not 27.3.
