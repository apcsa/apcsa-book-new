In BlueJ, every class is represented as a box, with arrows showing the relationship between classes.

Now, let's take a look at the content of the GameBoard project 1. As the screenshot shows, BlueJ shows a project as a collection of rectangles with arrows.

Each of the rectangles represents a class in the project. In the GameBoard project there are two classes, the GamePiece class and the GameBoard class.

The arrows between the rectangles show how the classes relate to one another. If there is an arrow pointing from one rectangle to another rectangle, it means that the class corresponding to the first rectangle uses the class corresponding to the second rectangle. This means that there is some behavior or instance variable in the GameBoard class that makes, refers to, or changes a GamePiece object.
