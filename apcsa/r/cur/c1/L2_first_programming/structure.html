Let's examine this class. First, notice the overall structure. Like other classes we will see in this course, it starts with the words "public class", followed by the name "CandyJar", followed by a left brace, followed by the "guts" of the class, followed by a right brace at the end of the code. 

public class CandyJar {

   ... the guts of the class ... 
	
}

Class names in Java, by convention, start with a capital letter. For a class name that really stands for two or more words, there are two common conventions:

  1. Capitalize all the words, then concatenate them (i.e. glue them together). For example, the words "candy jar" would be capitalized and then concatenated to form "CandyJar".
  2. Separate the word with an underscore character ("_"). That would result in the name "Candy_jar".

We will use naming convention #1 in this course. Thus, the name of our class is CandyJar, not Candy_jar.
