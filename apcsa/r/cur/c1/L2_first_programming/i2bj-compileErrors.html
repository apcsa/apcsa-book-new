Open the GameBoard class in the BlueJ editor. Change one of the occurrences of the word myDisplay to yourDisplay. Now try compiling the class by clicking on the Compile button at the top of the editor window.

What's going on here? The line that we just changed is now highlighted in yellow. Also, notice a message has appeared at the bottom of the editor window that reads: "cannot find symbol - variable yourDisplay".

BlueJ is trying to tell us that the compiler is having trouble understanding the line highlighted in yellow. This typically means that you've made a mistake at that line. The error message is generated by the compiler and it tries to be as informative as possible about what exactly it is that it doesn't understand. In this example, the changes that we made introduced a compile-time error. Change it back and compile the class again.

The compiler will help us catch a large number of errors, from simple typos to incorrect calls on objects' methods.
