So far, we've been creating objects of classes in the project GameBoard by right-clicking the rectangles. As you know, every class is defined by some source code, which is the text that defines the state and behavior of the class. In BlueJ, you can view the source code of a class by double-clicking the rectangle corresponding to the class or by selecting the Open Editor function from the class's popup menu.

Double-click the GameBoard class, and an editor window will appear. The editor window shows the variables that represent the state of the GameBoard class, and the methods that represent its behaviors. When you create your own classes, you have to provide variables and methods using the source code editor.

In Java, the name of the class must match the name of the file containing the definition of the class. In our case, the name of the class is "GameBoard" and the file containing its implementation is "GameBoard.java".
