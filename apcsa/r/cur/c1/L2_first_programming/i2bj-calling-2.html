Recall the discussion of parameters, pieces of information that let us write more general methods. Here's how a method with a parameter is called in BlueJ.

Make an object of the GamePiece class, and call it piece1. It should appear in the object bench. Here's a slide show depicting the following steps if you get confused: 1

To make your object visible in the window associated with gameBoar1, we need to add the piece to the board. Adding a piece is a behavior of a GameBoard, rather than a GamePiece. Right-click on the gameBoar1 object in the object bench, and select the addPiece method.

A dialog box will open up and you will be asked to type three things, because the addPiece method requires three parameters.

  1. The first is the piece—an object instantiated from the GamePiece class —to add. Provide the name of the piece to add in the top text box; your GamePiece object is named piece1, so type that in the first slot.
  2. The other two parameters indicate the location where this piece is to appear on the board. Enter 1 for the row number and 1 for the column number.

Click OK, and you have added piece1 to gameBoar1. Call the showBoard method, then look at gameBoar1's window and check that the piece was added.
