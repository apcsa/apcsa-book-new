As we just noted, in any arithmetic operation involving both a double and an int, the int gets converted to a double before the operation is performed. The conversion conceptually has the result of adding ".0" to the end of the integer.

Essentially the same thing happens in an assignment of an int right-hand side to a double variable: 

   double x;
   x = 25;      // x now contains 25.0

Assigning a double value to an int variable, however, produces a compiler error: "Possible loss of precision" in BlueJ. The compiler is protecting us from accidentally throwing away part of the double value—the part after the decimal point—even if it's zero. We'll see later in the course how to get around this limitation.
