You may be wondering how much a Refillable object can hold. Some experimentation produced the following results:

  1. Call the Refillable constructor with argument 2000000000, then call currentAmount. The object holds at least 2000000000.
  2. Now call refill with argument 1000000000, and call currentAmount again. It says -1294967296.

How can adding two positive values produce a negative value? Something is wrong.
