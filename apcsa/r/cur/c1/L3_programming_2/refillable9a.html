The name Refillable isn't a great one, but what name might be better? Consider the name Quantity. When we call something a "quantity", we imply that it is unchanging.

Below are the instance variables and methods of the Refillable class. Select those that do not belong in a class that represents an unchanging quantity of something:

myAmount (the instance variable)
No, a class representing a quantity would need a way to keep track of how large a quantity it is, certainly.
Refillable (the constructor method) which would now be named Quantity
No, a "Quantity" class would need a constructor to initialize its amount.
useUp (a method)
Right: the name "Quantity" doesn't imply an amount that can change, and useUp is a method to decrease the amount.
refill (a method)
Right: the name "Quantity" doesn't imply an amount that can change, and refill is a method to increase the amount.
currentAmount (a method)
No, a "Quantity" class would most likely need a way to tell user, or another class, how big the quantity that it represents is.
