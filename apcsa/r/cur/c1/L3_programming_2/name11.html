The caseFixed method has a String parameter. In earlier examples, we used parameters as a mechanism to get information into a method from outside. Here, the information needed by caseFixed—namely, which word in the name is to be "fixed"—is already available to the method, through the instance variables.

We could have coded the Name methods as follows (this class contains three very similar "fix..." methods):

// Note: This code is not good style
   public class Name {

   private String myFirst;
   private String myMiddle;
   private String myLast;
   
   public Name (String first, String middle, String last) {
      myFirst = first;
      myMiddle = middle;
      myLast = last;
      fixFirst ( );
      fixMiddle ( );
      fixLast ( );
   }
   
   public String lastFirst ( ) {
      return myLast + ", " + myFirst + " " + myMiddle;
   }
   
   public String fullName ( ) {
      return myFirst + " " + myMiddle + " " + myLast;
   }
   
   // bad style
   private void fixFirst ( ) {
      myFirst = myFirst.toLowerCase().trim();
      myFirst = myFirst.substring(0,1).toUpperCase() + myFirst.substring(1);
   }
   
   // bad style
   private void fixMiddle ( ) {
      myMiddle = myMiddle.toLowerCase().trim();
      myMiddle = myMiddle.substring(0,1).toUpperCase() + myMiddle.substring(1);
   }
   
   // bad style
   private void fixLast ( ) {
      myLast = myLast.toLowerCase().trim();
      myLast = myLast.substring(0,1).toUpperCase() + myLast.substring(1);
   }
}
