Inside a computer, it is more convenient to use a base that's a power of 2, for example,

binarybase 2"digits" are 0 and 1
octalbase 8"digits" are 0, ..., 7
hexadecimalbase 16"digits" are 0, ..., 9, A (representing 10), B (representing 11), C, D, E, F (representing 15)

For example, the value that 45 represents in base 10 is represented in binary as 101101, in octal as 55, and in hexadecimal as 2D, since 

45 base 10 = 1 x 25 + 0 x 24 + 1 x 23 + 1 ? 22 + 0 ? 21 + 1 ? 20
    = 5 x 81 + 5 x 80
    = 2 x 161 + 13 x 160
