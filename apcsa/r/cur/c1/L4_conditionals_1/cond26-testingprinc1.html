The most obvious principle of program testing is that test values must "exercise" every statement in the program, since any statement that's not tested may contain a bug. Our tests of the Refillable class basically did that, covering the cases in useUp and the cases in refill. As in the testing of Refillable, exercising all statements may require more than one run of a program.

Recall the leap year tester from an earlier exercise:

    if (year % 400 == 0) {
        leapDay = 1;
    } else if (year % 100 == 0) {
        leapDay = 0;
    } else if (year % 4 == 0) {
        leapDay = 1;
    } else {
        leapDay = 0;
    }

The code contains four cases, exactly one of which is executed for any particular value of year. Thus we must test this code with at least one year value per case, so at least four values of year are needed for testing:

  • a year that's divisible by 400;
  • a year that's divisible by 100 but not by 400;
  • a year that's divisible by 4 but not by 100;
  • a year that's not divisible by 4.

Principle 1 by itself is insufficient, however. Here's an example; 

    leapDay = 0;
    if (year % 4 == 0) {
        leapDay = 1;
    }
    if (year % 100 == 0) {
        leapDay = 0;
    }
    if (year % 400 == 0) {
        leapDay = 1;
    }

A year value of 2000 causes all the statements in this program segment to be executed.
