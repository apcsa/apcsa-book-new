We saw that there are often several ways to code a set of conditions:

  • consecutive conditional statements
  • cascaded conditional statements
  • nested conditional statements

The program logic can get quite complicated. One way to get evidence that the code works correctly is to test it. Thorough testing includes making sure that every statement gets executed; if a statement isn't executed, we have no evidence of its correctness.

Conditional statements complicate things even more. Where practical, test cases should exercise every path through the program. Use of a "tester class" can simplify the testing process.

Boundary values—values that are as large or as small as possible—are good candidates for tests. These address typical errors that programmers make, such as off-by-one errors.
