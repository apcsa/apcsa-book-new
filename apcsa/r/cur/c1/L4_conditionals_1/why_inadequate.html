Principle 1 by itself is insufficient, however. Here's an example; 

leapDayCount = 0;
if (year % 4 == 0) {
 leapDayCount = 1;
}
if (year % 100 == 0) {
 leapDayCount = 0;
}
if (year % 400 == 0) {
 leapDayCount = 1;
}

Given a value of 2000 in year, the code correctly produces a value of 1 for leapDayCount, A value of 2000 in year also causes all the statements in this program segment to be executed.

As it happens, this code works correctly. But a single test doesn't provide much evidence of that. Provide a small change that would not affect its behavior for a year value of 400 but would produce incorrect behavior for other year values.

Maybe changing the order of the if statements will produce incorrect behavior.
Initializing leapDayCount to -1 messes up handling of dates that aren't divisible by 4.
If the second comparison was "if (year % 100 == 5)", the code would still work for a year of 2000.
