We've looked at various ways to arrange tests to perform a given computation.

Choosing one way over another is largely a matter of personal preference, in my opinion. Ha ha.

Nested conditions are generally tougher to understand than conditions arranged in a cascading form. Compare the earlier (equivalent) examples: 

/* Cascading conditionals
 *  are easy to follow: */

if (temp > 80) {
    comfortLevel = "too hot";
} else if (temp < 50) {
    comfortLevel = "too cold";
} else {
    comfortLevel = "comfortable";
}

/* Whereas nested conditionals can
 *  quickly become confusing: */

if (temp >= 50) {
    if (temp > 80) {
        comfortLevel = "too hot";
    } else {
        comfortLevel = "comfortable";
    }
} else {
    comfortLevel = "too cold";
}
