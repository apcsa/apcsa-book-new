In order to thwart attempts to break a Refillable object, we will insert checks for illegal or not valid argument values. A typical pattern will be the following:

If argument is not valid, then print an error message.
Otherwise perform the appropriate action or computation.

Printing an error message will be done with the method System.out.println. (There are other ways to deal with invalid arguments that we'll learn about later in the course, but printing an error message will do for now.) Checking for an invalid argument is done with conditional statements, which we'll learn about shortly.
