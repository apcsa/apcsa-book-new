The programming analog to misusing an object is giving its methods invalid argument values.

We've previously seen that BlueJ will not allow some invalid arguments. For example, BlueJ will report an error if we try to pass a String to an int parameter.

Open up the current BlueJ project, and try supplying invalid arguments to the Refillable constructor. (This project is located at /lesson4/Refillable version2 on your computer).

In particular, try to provide an invalid argument value for the Refillable constructor—one that causes the object to be "broken"—for which BlueJ would not produce an error message.
