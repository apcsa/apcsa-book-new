A broken Refillable object

Hopefully, you see from the previous step that passing a negative argument value to the Refillable constructor allows the object to be created without an error. Explain how a Refillable object constructed with this kind invalid argument is "broken".

Although the Refillable object is constructed, it will fail when we call increase or decrease on it.
It is broken because we passed in an initial amount less than zero -- it doesn't make sense to have a refillable start with a negative amount
