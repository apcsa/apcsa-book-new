Suppose that a Refillable object has been constructed with a value of 27 for myAmount, and no other Refillable methods have been called. Which of the Refillable methods that do not take an argument would cause an error when called?

currentAmount
Nope. currentAmount is simply returning the value of a state variable. This cannot cause an error.
averageUse
Right! averageUse will cause an error because myUseUpCallCount will equal zero, and averageUse uses that variable to divide by. Java will complain of a "divide by 0" error.
