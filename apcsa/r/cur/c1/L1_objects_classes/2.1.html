In programming, we distinguish between the concept of an object— represented by a description of the object's behaviors and state—and the object itself. We hinted at that distinction previously. For example, the description of an alarm clock's behaviors and state isn't the same as an actual clock. There can be many actual clocks, but only one summarizing description. In programming, we call this concept or summarizing description a class. Particular things that fit the description are instances of the class. Instances all have the same behaviors, but usually have different state information. The state variables for a class or object are referred to as instance variables.

We've discussed classes already, describing clocks, toasters, cell phones, and MP3 players. Here are some more real-world examples.

real-world "class" real-world instances

A shoe class. Its state information includes size, color, and style. A simple behavior would be to communicate its size to someone.

An online restaurant reservation service. Its behaviors include making a reservation, cancelling a reservation, and determining what restaurants have tables available at a given time and date. Its state information includes the collection of upcoming reservations people have made and the set of restaurants at which one can make a reservation.

A song class. Among its behaviors is playing itself. Its state information includes the length in seconds, the type of music, and the artist.
