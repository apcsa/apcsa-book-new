Each variable has a name and a value.

Behaviors are actions that the object can perform; state consists of information (variables) that the object can remember.

An alarm clock has to be pretty good at remembering when it will do the "ring the alarm" behavior, or it would be a very poor alarm clock. Each object, either in the real world or in programming, needs to have a memory.

The way to have memory in object-oriented programming is to create a set of information items associated with a programming object. We call each of these a variable.

A programming object can keep track of many different pieces of information by containing many different variables. We refer to each variable by its name. Each variable will also have a value, the thing that is being remembered. Taken together, the set of information is called the object's state.

The alarm clock object should have a variable named alarm time, which could take a value such as "5 am", or "8:45 am", or basically any time. The behavior "ring the alarm" needs to know the value of that piece of information in order to work properly. Similarly, the behavior "set the alarm time" needs to change the value of that piece of information in order to work properly.

A rule of thumb in programming: behaviors generally don't change, but information often will.
