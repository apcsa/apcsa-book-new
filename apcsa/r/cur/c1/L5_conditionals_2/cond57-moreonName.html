Our updated Name class will be able to handle names of one (e.g. Madonna or Sting), two, or three components. Here's how.

  • Every Name object will have a non-empty first name.
  • Each missing word in the name will be represented by an empty string.

Here are some examples.

Name myFirst myMiddle myLast
Sting "Sting" "" ""
George Washington "George" "" "Washington"
Edgar Allan Poe "Edgar" "Allan" "Poe"

The result returned by the fullName method should only include non-empty words, with no blanks around a one-word name and a single blank between the first and last names in a two-word name.

For a one-word name, lastFirst should return only the first name, without a comma. For a two-word name, lastFirst should return the last name, followed by a comma and a blank, and then followed by the first name.
