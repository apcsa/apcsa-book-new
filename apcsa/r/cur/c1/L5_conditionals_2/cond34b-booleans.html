Modern programming languages are pretty smart. You may have noticed that, depending on whether the left side of a boolean expression is true or false, one may not need to evaluate the right side. For example:

Recall a previous step, where tempStr was set to "comfortable" or "uncomfortable" depending on the value of temp. 

if (temp >= 65 && temp <= 75) {
    tempStr = "comfortable";
} else {
    tempStr = "uncomfortable";
}

What happens when temp is assigned the value 55? The left-hand side of the statement evaluates to false and the right-hand side is skipped. Why?

if (temp >= 65 && ... ) {

If temp is less than 65, it doesn't matter what the "..." is. In order for the and ("&&") expression to evaluate to true, both its subexpressions must be true. In our example, as soon as the left subexpression evaluated to false, we knew the entire expression would be false. The process of cutting short the evaluation of a boolean expression is known as short-circuiting.
