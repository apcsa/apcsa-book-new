We will add two more constructors, one for a one-word name, the other for a two-word name. 

public Name (String first, String last) {
   myFirst = caseFixed (first);
   myMiddle = "";
   myLast = caseFixed (last);
}

public Name (String first) {
   myFirst = caseFixed (first);
   myMiddle = "";
   myLast = "";
}

Why not just retain the three-argument constructor and have the client object pass empty strings as arguments? Because we want to present an abstraction to the client by hiding the details of how names are actually implemented.
