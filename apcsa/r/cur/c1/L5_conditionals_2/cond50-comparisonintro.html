The Name class we worked with earlier had two big deficiencies. First, it could only handle names with a first name, a middle name, and a last name. Second, there was no easy fix for this problem; supplying a special string of some sort for (say) the middle name had undesirable side effects.

Conditionals allow us to fix this problem by having a special string for each missing part of the name, and by checking explicitly for this string when building the full name, the initials, etc.

Before we start modifying the Name class, we need to provide a few more details about how comparisons are made in Java.
