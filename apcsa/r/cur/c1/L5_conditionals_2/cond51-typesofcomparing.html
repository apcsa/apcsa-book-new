We have seen how the == operator is used to compare int values.

For example: 

int x = 10; int y = 10;
if (x == y) { 
   //do something
} else {
   //do something else
}

Here x == y "asks", does x match y? In this case, the answer is true—they are the same number.

However, == means something rather different when applied to objects. Consider this: 

Card c1 = new Card(9,2);
Card c2 = new Card(9,2);
if (c1 == c2) {
   //do something
} else {
   //do something else
}

To check that c1 == c2, where c1 and c2 each resulted from constructing a Card object, asks whether c1's object is the same object as c2's object. The answer here is false; c1 and c2 are not the same object even though their content, or instance variables, are the same. How do we compare these objects to see if their contents are same?

We rely on a method named .equals(...) to make the comparison for us. This gets a little complicated but it is fairly straightforward with strings. Two different String objects may contain the same characters in the same sequence, but comparing them with == will fail. Instead, we use the equals method in the String class. Given two String variables s1 and s2, the expression s1.equals(s2) has value true if s1 and s2 contain the same characters in the same sequence. It returns false otherwise.
