To make things more entertaining, Cam added "tiredness" to her Virtual Pet.   Her idea was to add a new instance variable, naming it tiredness, and adding Java statements to the behaviors so that the value of tiredness is changed.  Things, however, aren't working quite right. In fact, she can't even get her code to compile!

Open this project in BlueJ: "cam-v2" (in BlueJ, go to Tools>Open Current BlueJ Project to have BlueJ automatically open this project for you).

Then fix Cam's code.

You'll need to make sure you check the following:

  • Confirm that the code compiles!
  • If tiredness ever goes above 15, the pet should be set to the "asleep" mood.
  • The sleep behavior should always reset tiredness, while exercise and feed add 5 and 2 to tiredness respectively.
  • If the pet is both dead (hunger over 12) and asleep (tiredness over 15) then the pet should die. That is, death takes precedence over falling asleep!

Don't forget that inspecting an object is a useful way of checking on the values of its instance variables.
