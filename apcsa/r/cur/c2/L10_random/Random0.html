The values produced by Random aren't truly random, of course; they are produced by an algorithm. Here is how it works.

First, some terminology. Let's let r0 be the first value returned by a Random object, r1 be the second value returned, and so on. Each rk value is related to the previous value rk-1 by a formula: 

   rk = some of the digits in the quantity a * rk-1 + b

where a and b are specially chosen. The value from which r0 is calculated is called the seed. We can supply the seed as an argument to the Random constructor, or let the Java library supply a value by calling the 0-argument constructor.
