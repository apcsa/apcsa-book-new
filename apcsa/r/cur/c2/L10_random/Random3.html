For java.util.Random, a was chosen to be a large value. Also, instead of the rightmost digits, the value returned is chosen from digit positions further left in the product. More details are provided in the online Java documentation.

The computation also takes advantage of another quirk of Java arithmetic. The number of different values that can be stored in an int is limited; the maximum allowable int value is 231-1. One might expect some sort of error to result from trying to compute a value that's too big. (This is called overflow.) When we compute a sum or a product that's greater than 231-1, however, no error is signalled. This lets some unintended arithmetic errors go undetected, but simplifies the coding of classes like Random.
