Suppose you want to model a pair of dice. You might think of doing the following: 

import java.util.Random;

public class PairOfDice {

    private Random d1;
    private Random d2;
    
    public PairOfDice ( ) {
        d1 = new Random ( );
        d2 = new Random ( );
    }
    
    public void roll ( ) {
        System.out.println ("Roll: " + (d1.nextInt(6)+1)
                + " " + (d2.nextInt(6)+1));
    }
}

Recall, however, how the values returned by a Random object are computed. In the code above, we've let the Java system choose the seed; thus d1's seed and d2's seed are the same, as are their a and b values. Calling roll thus produces two equal die rolls!

The moral: use one Random object throughout a program. (One way to do this is via a static variable that's "owned" by all the objects in a class. We'll explore the use of static methods and variables in a few weeks.
