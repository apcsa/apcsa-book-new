The original xeyes program made the eye pupils follow the mouse as it was moved, not just when it was clicked. This makes things much cooler, so lets do it.

Make the pupil follow every move of the mouse, rather then just the position at which it is clicked. Save your work in the BlueJ project titled XEyes_motion.

If you think a bit about this, you should realize that there is no possible way to do it. When implementing mouseListener, you only get click events (and enter/exit events, which won't help). You are right!

What you need is a new kind of listener. Specifically, a MouseMotionListener. This is very much like the good old MouseListener, but requires different methods to be present:

  • mouseMoved: this method will be called everytime the mouse is moved while over the graphical object, even if by one pixel. Perfect!

  • mouseDragged: this will be called after a mouse has been pressed and moved. It will continue to be called everytime the mouse moves, until the mouse button is released. Interesting, but not useful here.

Like the MouseListener methods, these two are called with a MouseEvent object.

A class can implement two different interfaces at once. Simply list each interface, separated by commas, after the implements keyword. It should look like this: 

public class XEyes extends JPanel implements MouseListener,MouseMotionListener{

You will also need to be sure to add the mouse motion listener to the object in the XEyes constructor: 

public XEyes(){
	...
	addMouseMotionListener(this);
	...
}

And finally, implement the mouse motion listener contracted methods in the XEyes class:

public void mouseMoved(MouseEvent e){ ... }
public void mouseDragged(MouseEvent e){ ... }
