Try and see what happens when you click right inside an eyeball. Really, this looks terrible. The problem is that the movePupil method only draws the pupils along the outside rim of the eye.

When clicking inside the eyeball, we should draw the pupil directly on the click. Save your work in a BlueJ project titled XEyes_center.

To do this exercise, we'll only need to modify the Eye class. Specifically, we'll only need to change the movePupil method, which takes in the coordinates of the mouse click and sets the pupilX and pupilY instance variables, which hold the pupil's current position.

All you need to do is add an else if(...) case to the if-else chain in movePupil that tests if the mouse click was inside the eye and if so, sets pupilX and pupilY appropriately.

Use the following method to test if a point is inside the eye (derived from the formula of an ellipse):

public boolean inEye () {
        double dist = square(mouseX - eyeCenterX) / square(eyeWidth/2-pupilRadius) 
                 + square(mouseY - eyeCenterY) / square(eyeHeight/2-pupilRadius);
        return (dist <= 1);    
}

We'll leave you to write the method square, which returns a number times itself. Make sure you have square return a double, because the inEye method won't work with integer arithmetic!
