Putting the exact dimensions of each eye into the XEyes constructor is a hassle, especially when you use a different image. Let's make it so you can place an eye by clicking and dragging the mouse. Save your work in a BlueJ project titled XEyes_drag.

This will take several changes to the XEyes class, but none to the Eye class. Here are some hints on how to proceed:

  • If we are going to set the coordinates of the eye by clicking our mouse we need to remove the "hard-coded" coordinates passed arguments to the Eye constructor in the XEyes constructor. 

    public XEyes()
{
   ...
   eye_1 = new Eye(...);  //remove this line
   eye_2 = new Eye(...);  //remove this line too!
   ...	
}
    You will need to put a conditional in draw to only draw the eye if it has been created. But, how do you test for that?

    The easiest way is to set each instance variable to null initially. 

    Eye eye_1 = null;

    null is, essentially, the "empty" object.

    You can test for eye_1 or eye_2 being null in the paintComponent method by setting up these conditional: 

    if (eye_1 == null) {
   ...
   }
    if (eye_2 == null) {
   ...
   }
  • You can specify the size of the eye by checking the difference between the events of the mousePressed and mouseReleased methods. You did this in this lesson in an earlier step.

Two things you might want to add. First, you might want to ensure that there is a minimum size for an eye. Use a conditional just before you create the eye.

Second, you might create more eyes. Why stop at 2?
