The Eye class is quite a bit more complicated than, well, perhaps anything you've seen so far. Let's look at the big picture first.

The Eye object handles the drawing of one eyeball.

  • In order to keep track of what an eye looks like, objects from the Eye class have quite a few instance variables. The statement 
    int eyeX, eyeY, eyeWidth, eyeHeight, eyeCenterX,
 				eyeCenterY, pupilX, pupilY;
    is a shortcut for creating several instance variables of the same type. It defines 8 variables.
  • An eye knows how to draw itself, based on values of those instance variables, and does so through the draw method. Basically, an "eye" is three fillOval methods. Pay some attention to how the various instance variables define the dimensions that make up an eye.
