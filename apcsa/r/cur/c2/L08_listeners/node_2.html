The first step is to write the relevant mouse interaction (e.g., mouseClicked) method.

However, it is not enough to just include the method in your class. You have to tell Java that your class is both set up to deal with events and wants to receive those events.


How do you tell Java that your class can deal with events?

In a similar way to how we used graphics classes such as JPanel, we need to tell Java that our class can act like another class. We are going to use another special keyword in this case: the implements keyword. This is used in the class header, like extends. The thing that we will implementing, in this case, is a MouseListener. So, if we were writing a class called SimpleClicker that needed to deal with mouse events, we would write:

public class SimpleClicker implements MouseListener {
   ... 
}

What is MouseListener? It is capitalized like a class, but it is actually an interface, something somewhat different. You can think of an interface as a contract. That is, if you say a class "implements" an interface, that means you have to include certain specific methods in your class. For the MouseListener interface, that means the five methods in the previous page: mousePressed, mouseReleased, mouseClicked, mouseEntered, and mouseExited. Easy! (Incidentally, you can look in the online Java documentation to find which methods you have to provide when using an interface in the Java class library.)

We'll learn more about interfaces when we get to the unit on inheritance.
