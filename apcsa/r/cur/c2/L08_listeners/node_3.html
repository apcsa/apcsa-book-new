Since we have a graphical object (the JPanel, in the case of SimpleClicker), it makes sense to combine the mouse handling with graphic output.

As with the work we did in the last lesson, this means including a paintComponent callback method. There is an extra wrinkle, however: we are likely to need to change the graphics in response to a mouse click. That is, we need to have some control over when the paintComponent method is run.

We aren't allowed to call paintComponent directly; anyway, how could we make the Graphics object that it takes as an argument? Rather, we can call a method repaint, which takes no arguments, and forces Java to immediately update the screen.

Because we aren't calling paintComponent directly, we can't pass any information to it. Therefore, if we want to use something known in the mouse event—say, the coordinate where the mouse was clicked—that information has to be stored in an instance variable. That is, the mouse method stores the information, and then the paintComponent method reads it.
