Now, open up the source code for the XEyes class, which is the main starting point for the program.

As with many of the classes you have seen in this lesson, XEyes imports the graphics libraries, implements MouseListener, extends a JPanel, and has a JFrame stored in an instance object. Objects from this class, then, represent the program's window and the area where the graphics are drawn.

We know that it will have a paintComponent method, a constructor, and implement all the contracted MouseListener methods.

To handle the mouse input, the XEyes class implements mouseListener, meaning it will necessarily have the five mouse handling methods. Based on what you found out from playing with the program, you can assume that the mouseClicked method will figure out what the new position of the pupil is.

The constructor calls a helper method to do more work to get an image based on the string fileName. (The class File is imported from java.io.*).

The image comes from a file inside the images directory in your BlueJ project. Later, you can use your own images or ones you find, if you put them in that directory.

Note also that the XEyes class has two instance variables of the class Eye. These Eye objects are constructed with arguments very similar to drawRect, specifying an origin and a size.

The paintComponent method first draws the image, and the asks the eyes to draw themselves.
