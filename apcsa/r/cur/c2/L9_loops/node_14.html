This lesson introduced the while loop, a mechanism for repeating the execution of a given sequence of statements. A while loop has two parts: the test (a boolean expression), and the loop body (a sequence of statements). It is executed by first evaluating the test. If the result is false, the loop terminates; execution continues with the statement following the loop body. If the result is true, it executes all the statements in the body, then checks the test again. If that result is false, the loop terminates. Otherwise it executes all the statements in the body again, then checks the test again, and so on.

Loops introduce a significant level of complexity to programming. In particular, it is easy to have bugs in a loop that are difficult to find. Thorough testing is a good idea, using test data that exercise "typical" behavior as well as boundary cases—data for which the loop should execute 0 times, the minimum number of times, and the maximum number of times. It is often a good strategy to design the tests before the code!

Any reasonable programming environment comes with a debugger that lets you monitor the progress of a program. Debuggers let you do the following:

  • Set breakpoints, places in the program where the debugger will stop.
  • Single-step the program by executing the next statement and then stopping the program again.
  • Examine variable values.
