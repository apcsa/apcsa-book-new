Activities in this lesson introduced the process of setting up a window and drawing in it. A "window" in Java consists of two objects: a JPanel, on which drawing is done, and a JFrame, which acts something like an easel holding the JPanel. The easel/drawing relationship is illustrated below.

The JPanel and JFrame classes are supplied in class libraries that are distributed with all Java programming environments. The libraries have names; a library class is accessed by giving the import keyword followed by the name of the desired library. Here, we need to import two libraries: 

   import java.awt.*;
   import javax.swing.*;

The import statement must appear at the top of a program.

The import statement allows us to access a class. In contrast, the extend keyword lets us use the class, typically by supplying some methods of our own that augment or modify those of the extended class. (This technique is called inheritance and is covered in (much) more depth starting in lesson 16.) The code we worked with in this lesson all involved providing a class that extends the JPanel class: 

   public class Greetings extends JPanel {

Setting up the connections between JPanel and JFrame is somewhat complicated, but is pretty much consistent from program to program. We start by constructing a JFrame, then call some of its methods to

  • supply its width and height dimensions,
  • specify what to do when the "close" box is clicked,
  • add the JPanel to the JFrame, and
  • and display the window on the screen.

Here's the code: 

   JFrame easel = new JFrame();		
   easel.setSize (myWindowWidth, myWindowHeight);
   easel.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
   easel.add (this);
   easel.setVisible (true);

Finally, we got to do some drawing. This is done in the paintComponent method. The paintComponent is an example of a callback method. It's not called directly in our code; instead, it is called by the Java system when some action requires the window to be redrawn. paintComponent has a single argument, a Graphics object whose methods include facilities for drawing lines, text, and shapes.

Relevant Graphics methods are drawString, drawLine, drawRect, fillRect, drawOval, and fillOval. All take coordinates for a position in the drawing window as arguments. Coordinates are all nonnegative—the upper-left corner of the window has coordinates (0,0)—and increase as we move down or to the right. (Refer to the Java online documentation for further details of how these work, and for other methods of the Graphics class.)
