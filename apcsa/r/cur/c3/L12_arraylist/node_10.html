Some of the classes we studied earlier made use of several instance variables to represent a collection of information. Examples include

  • myFirst, myMiddle, and myLast in the Name class;
  • racer0, racer1, and racer2 in the DotRacing class; and
  • door1isClosed, door2isClosed, and door3isClosed in the Stage class in the Monty Hall program.

Code to process these collections is somewhat clumsy. For instance, consider the DotRacing whoMoves method: 

public Dot whoMoves ( ) {
   int moverChoice = randGen.nextInt (2);
   if (moverChoice == 0) {
      return racer0;
   } else if (moverChoice == 1) {
      return racer1;
   } else if (moverChoice == 2) {	
      return racer2;
   }
}

With more doors, the code would be even more awkward. Ideally, there would be a way to use a single statement, something like

Use moverChoice to access the racer___ variable, and return its contents.
