ArrayList objects solve this problem. An ArrayList (found in the java.util library) represents a sequence of elements, all the same type, in which each element is accessed by its position in the sequence (a nonnegative integer). For example, consider a recoding of whoMoves that uses a three-Dot ArrayList named racers: 

public Dot whoMoves ( ) {
   int moverChoice = randgen.nextInt (3);
   return racers.get (moverChoice);
}

As with any object, setting up an ArrayList includes declaration, for example, 

   ArrayList racers;

and initialized, 

   racers = new ArrayList ( );

The elements of the ArrayList must also be initialized.

ArrayLists have many methods (go to the online Java documentation to discover them). We worked with several, listed below. (Assume that values is an initialized ArrayList.)

values.add (element) Put the argument at the end of the sequence
values.add (position, element) Insert the second argument into values at the given position, moving subsequent elements up one position to make room.
values.size ( ) Return the number of elements in values.
values.get (position) Returns the element at the given position in values.
values.remove (position) Remove the element at the given position from values, moving subsequent elements down one position.

Legal position values range from 0 to size ( ) - 1. A common bug is to use an incorrect position, producing an "array index out of range" error.
