In programs we've written so far, every piece of information had to be stored in a separate variable. In the Name class, we needed three instance variables to store the three components of a name. To keep track of three dice, we needed three DieRollGenerator variables. And so on.

Upcoming activities will introduce a collection class called an ArrayList. Such a class is appropriate whenever we have a group of elements, all the same type, that we want to store collectively. Some real-world examples are the following:

  • a book (elements are the individual pages);
  • a parking lot (elements are the individual parking spaces);
  • a library (elements are the individual books).
