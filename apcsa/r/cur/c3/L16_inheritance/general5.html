In the Monty Hall simulation, we compared the strategy of always switching doors with the strategy of never switching.

A version of that program using inheritance could use a Player class with a chooseDoor method, along with subclasses SwitchingPlayer and StayingPlayer that override the chooseDoor method to implement the switching and staying strategies.

Some board games have pieces that share some attributes but move differently. For example, an uncrowned checker piece moves differently from a king. The features common to all pieces could be included in a superclass that would be inherited by a class representing uncrowned pieces and another class representing kings.
