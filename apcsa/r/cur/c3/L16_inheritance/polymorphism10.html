Once the compiler is satisfied that a move method is available for an object, the runtime system then selects the most specialized move method to call. For a Point, it uses the Point move method. For a TracedPoint, it uses the TracedPoint move method.

Why did it work? Because the compiler noted that a toString method is defined in the Object class, and was satisfied.

A mantra to help understand this: an object always remembers how it was constructed, and chooses a method from the class of which it is an instance.
