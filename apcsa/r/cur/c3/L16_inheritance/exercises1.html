Some classes provide "setter" methods. A useful debugging aid is to override a setter method to produce informative output at each time that the object's state changes.

We've already worked with the Point class in the java.awt library. Its setter method is named move; the call 

   p.move (27, 13);

changes the x coordinate to 27 and the y coordinate to 13 in the Point referenced by p. It also provides getX and getY access methods.

Given below is the framework for a TracedPoint class intended to intercept calls to move and print information about the pre- and post-change state of the Point along with doing the move. If the TracedPoint object named p currently represents the point (3,5), then the call 

   p.move (9,8)

should print a message like 

   Point change: was (3,5), is now (9,8)

as well as changing the point's value.

Now you should complete and test the following framework:

import java.awt.*;

public class TracedPoint extends Point {

    public TracedPoint (int x, int y) {
        super (x, y);
    }

    // Your move method goes here.
}

Take advantage of the superclass's methods and variables as much as possible rather than reinventing the wheel. In particular, your solution shouldn't have any instance variables.
