As you may have noticed, it's possible to pass a subclass instance where a superclass instance is expected. In order to compare, say, the Unforgiving player with the TitForTat player in the Prisoner's Dilemma simulation, we created an instance of Unforgiving and an instance of TitForTat and passed them as arguments to the Simulation constructor, which is expecting to receive two Player objects. Not only will this be accepted by the compiler, but the compiled code will use the cooperates methods of Unforgiving and TitForTat.

This facility is called polymorphism. The word "polymorphism" comes from the Greek words for "many" and "forms". In the context of object-oriented programming, it means that a given object can be regarded either as an instance of its own class, as an instance of its superclass, as an instance of its superclass's superclass, and so on up the hierarchy.
