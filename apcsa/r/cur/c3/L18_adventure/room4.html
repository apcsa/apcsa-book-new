Text adventure games need goals, and ways for the player to reach those goals (and end the game). In this step, you'll make a new subclass of Room that ends the game when you enter it.


First, how does a method end the game? Examine the play method in the World class. The loop stops when the isPlaying boolean instance variable is set to false. That is a private instance variable (as it should be), so other objects won't be able to change it. Rather, the World method gameOver lets any object turn the game off.


So, the task is to create a new class that extends Room and calls gameOver when it is entered?


Precisely! Do it now, and add a room (a bathroom?) where the player can find the inner peace she seeks. (The next page has a few tips for you).
