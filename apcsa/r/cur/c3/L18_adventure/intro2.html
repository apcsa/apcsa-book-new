Before looking at the Java code that we'll start with, let's look at the BlueJ object diagram and talk about the classes in the program:

There are three main areas to this program, although each area contains two classes:

  • Getting the typed input from the user, and interpreting it. This functionality is in the InputGather and Command classes at the right of the diagram. Notice that these classes are related through inheritance.
  • Representing the rooms and the connections between them. The Room and Path classes contain these functions. We'll spend some time extending the Room class to make more interesting rooms.
  • Controlling the general play of the game. The World and Player class set up the map for the game and contain the basic loop in the game: get a command from the user, print out information about what happens, and repeat.
