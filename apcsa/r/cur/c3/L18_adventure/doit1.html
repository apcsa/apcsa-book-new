Here are some ideas about interesting things you can do...


  • Make and item that has states that change as the user interacts with them (e.g., a balloon that pops).
  • Make a room with an exit that only becomes visible when a certain item is held by the player.
  • Make an exit of a room a "random transporter". When the player uses it, it puts him in a random room. You'll want to subclass Path for this one (think about which method you want to override...). And, you'll have to change the TransporterPath constructor so that it stores a reference to the World object, so it can choose a room.
  • Make an item that can be used with another item to make something totally different.
