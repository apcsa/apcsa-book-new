Examine the Room class in the Adventure_v0 project.

It is a relatively simple class. It has three instance variables:

  • name, a string
  • description, another string
  • exits, an ArrayList of Path objects. (Objects of the Path class connect two rooms with a particular direction. The direction associated with a Path is the name or direction of the exit from the current room.)

and only a few methods. The methods that control movement between rooms include:

  • enterRoom, which prints the Rooms description and then returns a boolean value of true, indicating that the player successfully entered the room.
  • exitRoom, which returns the boolean value true, indicating that the player is allowed to exit the room.
  • tryToExit, which gets passed a String direction. If the direction string matches one of the exit directions in the room, and the exitRoom method returns true, it then figures out the next room based on the Path objects travelDestination method.

And, all of this is controlled by the actionTravel method in the Player class.
