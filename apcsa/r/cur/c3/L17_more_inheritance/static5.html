Another application that requires a static variable is the maintenance of a counter that keeps track of how many objects in a class have been instantiated. Such a counter might be useful in a card game, where we would want to make sure that a player isn't allowed to cheat by generating new cards beyond those already in the deck.

Here's part of a Card class with a card counter and NUMSUITS and NUMRANKS constants. Note the use of "our" in the static variable name rather than "my", to indicate that the variable is shared among all the cards. 

public class Card {

   public static final int NUMSUITS = 4;
   public static final int NUMRANKS = 13;
   private int myRank;
   private int mySuit;
   private static int ourCardCounter = 0;
   private static final int MAXCARDS = 52;
   
   public Card (int rank, int suit) {
      if (rank >= 0 && rank < NUMRANKS) {
         myRank = rank;
      } else {
         ... // signal error
      }
      if (suit >= 0 && suit < NUMSUITS) {
         mySuit = suit;
      } else {
         ... // signal error
      }
      ourCardCounter++;
      if (ourCardCounter > MAXCARDS) {
         ... // signal error
      }
   }
   
   ...
}
