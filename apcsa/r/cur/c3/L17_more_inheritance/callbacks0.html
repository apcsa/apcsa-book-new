It is common in Java for a method we provide to be called not by our own code but by someone else's. Two examples we've seen in earlier activities are the toString and mouseClicked methods. toString is rarely called directly. More likely, it is called by code that needs to produce a people-readable representation of an object, for instance, the System.out.print method. mouseClicked, as we have seen, is called by the Java system when the mouse is clicked in a suitably initialized drawing window. The situation where we call a method of a library class and another of our methods gets called as a result is referred to as a callback.

Another frequent participant in a callback is the equals method. For example, the ArrayList contains method calls equals to determine if the argument to contains is an element of the list. We will experiment with the equals method in the context of a Measurement class that represents measurements in feet and inches.
