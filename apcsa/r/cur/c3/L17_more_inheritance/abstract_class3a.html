But why can't you instantiate an abstract class?

Think of it this way: You borrowed a recipe book from Bolt, but his book only has a table of contents.
If you want to make Screw and Cable Salad, can you make it from just referring to the book? Probably not, because there are no instructions at all!

Next time I'm borrowing the recipe book from Twitterton.

Basically, an abstract class is specifying a set of capabilities that its subclasses are to provide. We use an abstract class when we don't want to define a method—possibly because any definition would be too general or inappropriate in some other way—but we do want our subclasses to define the method.
