Java has a special construct for an abstract class with all abstract methods and no instance variables. It's called an interface. We don't extend an interface; instead, we implement it, using the keyword implement. In doing so, we promise to provide definitions for all the methods named in the interface.

You've seen this already in activities that handled mouse clicks. Your class headers contained the phrase "implements MouseListener", thus promising to provide implementations for five methods: 

void mouseClicked (MouseEvent e) 
void mouseEntered (MouseEvent e) 
void mouseExited (MouseEvent e) 
void mousePressed (MouseEvent e) 
void mouseReleased (MouseEvent e)

Recall that we generally provided only empty method bodies for mouseEntered and mouseExited. That's enough to satisfy Java's requirements.
