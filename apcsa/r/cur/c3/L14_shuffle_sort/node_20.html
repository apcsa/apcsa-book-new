We also considered the efficiency of the two sorting algorithms. There are various ways to measure efficiency. Some measurements may vary, depending on the type of computer or version of Java we're using. A quantity that will be the same no matter what computer we use is statement counts.

The most interesting kind of statement to count in a sorting algorithm is the number of comparisons made to sort the collection of values. We produced this count by adding an instance variable to the outranks method in the sorting code, and found two items of interest:

  • For a given collection size, selection sort always involves the same number of comparisons, no matter what the arrangement of the elements of the collection.

  • For a given collection size, insertion sort does the same number of comparisons as selection sort when the elements are in reverse order. However, when the elements are already in order, insertion sort does substantially fewer comparisons.
