(This problem is adapted from a free-response question on the 1988 AP CS exam.)

Consider a class named Image that represents an black-and-white image. Its instance variable is a rectangular two-dimensional array of nonnegative integers, which we'll call myImage. each integer representing one of the pixels in the image. (The darker the pixel, the bigger the corresponding array element, and vice versa.)

Provide a public void method named zoomOut that expands myImage by an amount specified by its positive integer argument, which we'll call factor. In particular, if myImage starts with r rows and c columns and if the value of factor is k, then myImage is replaced by a two-dimensional array with k*r rows and k*c columns, where each element e of myImage is replaced by a k-by-k array, all of whose elements are e.

Because each Image takes up a significant amount of memory, you cannot assume that a second Image object would fit in memory. Thus, you will receive no credit if you make use of any array variable other than the one passed into the procedure.

You may find it helpful to write and call an auxiliary procedure that assigns a value to every element of a rectangular portion of a two-dimensional array.
