To understand how the revised sumHelper works, we repeat the steps covered earlier:

  1. Examine the base case. There's only one base case, and it returns the correct value for an empty interval.
  2. Try arguments that are one recursive call away from a base case. This would be a one-element interval (for which left == right), say, with left and right both equal to 4. It computes mid = (left+right)/2, which is 4, and adds that to the values returned by sumHelper (4, 3) and sumHelper (5, 4), which are both 0. So far, so good.
  3. Let's now try a three-element interval, namely [4,6]. mid is (4+6)/2, which is 5; the recursive calls are sumHelper (4, 4), which returns 4 as just described, and sumHelper (6, 6), which returns 6.

Do you see a pattern?
