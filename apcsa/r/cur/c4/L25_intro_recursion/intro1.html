Suppose we have a method named doIt that contains a call to itself that's always executed. What would happen? The initial call to doIt would

  • call doIt, which would
    • call doIt, which would
      • call doIt, and so on ...

Either this would go on forever, or it would crash somehow.

But now suppose that the call is guarded, so that it sometimes returns rather than calling itself. Execution might then proceed as follows:

  • call doIt, which would
    • call doIt, which would
      • call doIt,which would return
      which would return
    which would return

That's how we'll structure our own recursive code.
