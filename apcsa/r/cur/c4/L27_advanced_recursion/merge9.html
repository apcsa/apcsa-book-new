Why is merge sort relevant? Because it needs significantly fewer comparisons than insertion sort or selection sort to put the array elements in order.

For example, if N is the number of elements in the array to be sorted, both insertion sort and selection sort may require a number of comparisons proportional to N2, while merge sort needs only a comparison count proportional to N log N ("log" is the logarithm function).

This can make a big difference. For instance, let's say you have a computer that can compare one million pairs of numbers a second. If you need to sort one million numbers, merge sort could do this within a few seconds, while insertion or selection sort might take several days!
