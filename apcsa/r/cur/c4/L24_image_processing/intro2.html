We've provided a simple class for you to write methods within, and created another class to hide away most of the "ugliness" of converting image formats, saving to disk, and so forth.

Open the current BlueJ step.
(This is lesson24/ImageProcessor)

Of the two classes, you will work entirely in the RGBImage class. (Feel free to look inside the Helper class if you really want to).

Open the RGBImage class. Near the top you will see the central data objects of an image: 

    private int red[][];
    private int green[][];
    private int blue[][];

These three arrays carry the red, green, and blue values for each pixel. Thus, the color of the upper left pixel in the image is determined by red[0][0], green[0][0], and blue[0][0]. The size of these arrays will be set automatically by the image that you have loaded. (We won't be changing image sizes in this lesson, just manipulating the pixel colors).

Most of the methods in the RGBImage class are to help you create an instance based on an image in your BlueJ project folder, save your work to the file, and so forth. Reading the comment above the definition should make things clear. Your manipulation methods, as we'll talk about next, will be added to the end of the file.
